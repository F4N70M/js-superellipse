# Режимы работы

Библиотека предоставляет два режима применения суперэллипса: `clip-path` и `svg-layer`.  
Выбор режима влияет на производительность, поддерживаемые CSS-свойства и сложность DOM-структуры.

## Сравнение режимов

| Характеристика | `clip-path` | `svg-layer` |
|----------------|-------------|-------------|
| **Способ реализации** | CSS `clip-path` | Наложенный SVG-слой |
| **Дополнительные DOM-узлы** | Нет | Да (SVG + div-обёртка) |
| **Поддержка `background`** | ❌ (обрезается) | ✅ |
| **Поддержка `border`** | ❌ | ✅ |
| **Поддержка `box-shadow`** | ❌ | ✅ |
| **Поддержка градиентов** | ❌ | ✅ |
| **Поддержка `border-radius`** | ✅ (исходный, но переопределяется) | ✅ (переносится в слой) |
| **Производительность** | Высокая (только обрезка) | Ниже (рендер SVG + фильтры) |
| **Сложность** | Низкая | Высокая |

---

## Режим `clip-path`

Самый лёгкий и быстрый способ. Использует CSS-свойство `clip-path: path("...")` для обрезки элемента по форме суперэллипса.

### Особенности

- Не создаёт дополнительных DOM-узлов.
- Исходный `background`, `border` и `box-shadow` элемента **обрезаются** по форме суперэллипса (но не переносятся в слой, из-за чего могут выглядеть некорректно).
- Подходит для простых элементов без сложного оформления (например, изображения, иконки).

### Пример

```js
const element = document.querySelector('.image');
const controller = element.superellipseInit({ mode: 'clip-path' });
```

### Когда использовать

- Нужна максимальная производительность.
- Элемент не имеет фона, границ или теней.
- Элемент – это изображение, и его обрезка по форме – единственное требование.
- Элемент находится в анимированном или часто перерисовываемом контексте.

---

## Режим `svg-layer`

Полнофункциональный режим, создающий наложенный SVG-слой, который повторяет геометрию суперэллипса и отрисовывает фон, границы и тени.

### Как это работает

1. Исходный элемент получает `position: relative` и очищается от фона/границ/теней.
2. Создаётся внутренний `div` (`class="jsse--svg-layer--content"`), в который перемещаются все дочерние узлы.
3. Создаётся SVG-элемент с `position: absolute`, наложенный поверх внутреннего `div`.
4. SVG содержит:
   - Путь (`<path>`) формы суперэллипса.
   - Фильтры для теней (`box-shadow`).
   - Обводку для границ (`border`).
   - `foreignObject` с внутренним `div`, куда скопированы вычисленные стили `background` и `box-shadow`.
5. Все стили синхронизируются при изменениях.

### Особенности

- Поддерживает `background`, `border`, `box-shadow` в полном объёме.
- Поддерживает градиенты, сложные фоны (изображения, цвета).
- Требует больше ресурсов из-за рендеринга SVG и фильтров.
- Может влиять на события мыши (у SVG `pointer-events: none`, поэтому клики проходят сквозь).

### Пример

```js
const card = document.querySelector('.card');
const controller = card.superellipseInit({ mode: 'svg-layer', curveFactor: 1.2 });
```

### Когда использовать

- Элемент имеет фон, границы или тени.
- Нужно сохранить оригинальное визуальное оформление.
- Не критична производительность на очень большом количестве элементов.
- Требуется поддержка сложных стилей.

---

## Переключение режима

Вы можете переключить режим на лету у уже инициализированного элемента:

```js
// Переключение с svg-layer на clip-path
controller.switchMode('clip-path');

// Обратно
controller.switchMode('svg-layer');
```

При переключении:
- Старый режим уничтожается (виртуальные элементы удаляются, стили восстанавливаются).
- Новый режим активируется с текущими настройками (`curveFactor`, `precision`).

## Рекомендации

| Сценарий | Рекомендуемый режим |
|----------|---------------------|
| Изображения, аватары, иконки | `clip-path` |
| Карточки с фоном и тенями | `svg-layer` |
| Кнопки с градиентной границей | `svg-layer` |
| Анимированные элементы (много кадров) | `clip-path` |
| Сложные фоны (изображения, паттерны) | `svg-layer` |
| Мобильные устройства (экономия батареи) | `clip-path` |

---

## Известные ограничения режимов

### `clip-path`
- Тени и границы обрезаются, но не отображаются корректно.
- Некоторые браузеры могут отображать `clip-path` с артефактами при скруглении.

### `svg-layer`
- `border-style: double` не поддерживается (выдаётся предупреждение).
- `box-shadow` с `inset` не поддерживается (игнорируется).
- При большом количестве элементов (десятки и сотни) может снижать производительность.
- SVG-слой создаёт дополнительную вложенность, что может влиять на CSS-селекторы типа `:first-child`.

## Пример: сравнение визуального результата

### Исходный элемент с фоном, границей и тенью

```css
.card {
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
  border: 2px solid white;
  box-shadow: 0 10px 20px rgba(0,0,0,0.2);
  border-radius: 30px;
}
```

| Режим | Результат |
|-------|-----------|
| `clip-path` | Фон и тень обрезаются по форме, граница может выглядеть неровно. |
| `svg-layer` | Фон, граница и тень идеально повторяют форму суперэллипса. |

Выбирайте режим в зависимости от визуальных требований вашего проекта.
