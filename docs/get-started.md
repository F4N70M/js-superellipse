# Get Started с sj-superellipse

## Базовая инициализация

Самый простой способ — воспользоваться глобальной функцией `superellipseInit`.

### По CSS-селектору

```javascript
// Применить суперэллипс ко всем элементам с классом .box
const controllers = superellipseInit('.box', {
  curveFactor: 1.2,
  precision: 3
});
```

Функция вернёт массив контроллеров для всех найденных элементов.

### Для одного элемента

```javascript
const element = document.getElementById('hero');
const controller = superellipseInit(element, {
  mode: 'clip-path',    // режим через clip-path
  curveFactor: 0.8
});
```

### Для коллекции (NodeList или массив)

```javascript
const items = document.querySelectorAll('.item');
const itemControllers = superellipseInit(items, {
  curveFactor: 1.5
});
```

## Настройки (options)

| Опция         | Тип      | По умолчанию            | Описание |
|---------------|----------|-------------------------|----------|
| `mode`        | `string` | `'svg-layer'`           | Режим реализации: `'svg-layer'` (добавляет SVG-слой) или `'clip-path'` (использует CSS `clip-path`). |
| `curveFactor` | `number` | `0.5522847498` (константа G) | Коэффициент кривизны углов. Диапазон от **-2 до 2**. |
| `precision`   | `number` | `2`                     | Количество знаков после запятой в сгенерированном SVG-пути. |
| `autoResize`  | `boolean`| `true`                  | Автоматически отслеживать изменение размеров элемента. |

### Значения `curveFactor`

| Значение | Форма угла |
|----------|------------|
| `-2`     | Вогнутые прямоугольные углы |
| `-0.5523`| Вогнутые круглые углы |
| `0`      | Прямой скос |
| `0.5523` | Выпуклые круглые углы (четверть окружности) |
| `1`      | Классический суперэллипс |
| `2`      | Выпуклые прямоугольные углы |

Промежуточные значения дают плавный переход между указанными формами.

## Доступ к контроллеру и управление

После инициализации можно получить контроллер через геттер `.superellipse`:

```javascript
const controller = document.querySelector('.box').superellipse;
```

### Методы контроллера

- `switchMode(mode)` — переключает режим между `'svg-layer'` и `'clip-path'`.
- `setCurveFactor(value)` — изменяет коэффициент кривизны.
- `setPrecision(value)` — устанавливает точность генерации пути.
- `destroy()` — полностью уничтожает контроллер, восстанавливает исходные стили и удаляет обёртки.

```javascript
// Изменить кривизну
controller.setCurveFactor(1.8);

// Переключить режим
controller.switchMode('clip-path');

// Уничтожить
controller.destroy();
```

## Режимы работы

### `svg-layer` (по умолчанию)

- Внутри элемента создаётся абсолютно позиционированный `<svg>`.
- Фоновые стили (`background`, `border`, `box-shadow`) применяются к SVG-слою.
- Оригинальное содержимое оборачивается в `<div>` с `position: relative`, чтобы не перекрывать слой.

**Когда использовать:** когда нужно сохранить оригинальные стили элемента (например, текст поверх фона) и при этом использовать сложные фоны.

### `clip-path`

- Используется CSS-свойство `clip-path: path(...)`.
- Стили остаются на самом элементе, дополнительные DOM-узлы не создаются.

**Когда использовать:** когда важна производительность и минимальное воздействие на DOM, но при этом нет сложных фонов (например, если элемент — просто изображение или однотонный фон).

## Пример с полным управлением

```html
<div id="my-card" class="card">Содержимое</div>
```

```javascript
import { superellipseInit } from './api.js';

const card = document.getElementById('my-card');
const controller = superellipseInit(card, {
  mode: 'svg-layer',
  curveFactor: 1.2,
  precision: 3
});

// Через секунду изменим форму
setTimeout(() => {
  controller.setCurveFactor(1.8);
}, 1000);

// Ещё через секунду переключим режим
setTimeout(() => {
  controller.switchMode('clip-path');
}, 2000);
```

## Что нужно знать

- Элемент должен иметь явные ширину и высоту (заданные через CSS или атрибуты).
- При смене режима контроллер автоматически переносит отслеживаемые стили (`border-radius`, `background`, `border`, `box-shadow`, `position`) между элементом и SVG-слоем.
- Если элемент временно скрыт (`display: none`), обновления приостанавливаются и возобновляются после появления.
- При удалении элемента из DOM контроллер сам уничтожится (через `MutationObserver`).
