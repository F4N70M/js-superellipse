# FAQ, ограничения и требования

## Системные требования

- **Современный браузер** с поддержкой:
  - `ResizeObserver`
  - `MutationObserver`
  - `IntersectionObserver`
  - `CSS.supports()`
- **ES6+** (классы, модули, стрелочные функции)
- **Опционально**: поддержка CSS-селектора `:has()` для оптимальной работы hover-триггеров (если браузер не поддерживает, используется fallback).

### Проверенные браузеры

| Браузер | Минимальная версия |
|---------|-------------------|
| Chrome  | 64+ |
| Firefox | 69+ |
| Safari  | 12.1+ |
| Edge    | 79+ |
| Opera   | 51+ |
| iOS Safari | 12.2+ |

## Ограничения

### Режим `clip-path`
- **Не поддерживает** корректное отображение:
  - `box-shadow`
  - `border` (граница обрезается, но может выглядеть неровно)
  - сложные фоны (градиенты, изображения) – обрезаются, но не переносятся в слой
- При очень больших значениях `border-radius` может возникать небольшое искажение углов.

### Режим `svg-layer`
- **`border-style: double`** не поддерживается (библиотека выдаёт предупреждение в консоль).
- Производительность может снижаться при одновременной анимации большого количества элементов (десятки и сотни).
- Создаёт дополнительную DOM-вложенность (SVG + div), что может повлиять на CSS-селекторы типа `:first-child`, `:nth-child`.
- В старых браузерах (например, IE11) не работает из-за отсутствия необходимых API.

### Общие ограничения
- **CSS-правила из внешних источников (CORS)**: если таблица стилей загружена с другого домена и не имеет атрибута `crossorigin`, библиотека не сможет прочитать её содержимое для анализа hover-триггеров. В консоль будет выведено предупреждение.
- **`file://` протокол**: при открытии HTML-файла локально (без веб-сервера) доступ к стилям может быть ограничен политикой безопасности. Рекомендуется использовать локальный сервер (например, `npx http-server`).
- **Анимация `border-radius` через CSS-переходы**: работает, но может вызывать частые пересчёты пути. Для плавной анимации лучше анимировать `curveFactor` через `setCurveFactor()`.

## Часто задаваемые вопросы

### 1. Почему не работают тени или границы в режиме `clip-path`?

Режим `clip-path` только обрезает элемент по форме суперэллипса, но не переносит фоновые стили в отдельный слой. Если вам нужны тени, границы или сложные фоны, используйте режим `svg-layer`:

```js
controller.switchMode('svg-layer');
```

### 2. Как ускорить работу на странице со множеством элементов?

- Используйте режим `clip-path` (он легче).
- Уменьшите `precision` до 1 или 0.
- Избегайте одновременной анимации большого количества элементов.
- Включайте `debug: false` (по умолчанию) в production.

### 3. Поддерживаются ли вложенные суперэллипсы (элемент внутри другого)?

Да, библиотека корректно работает с вложенными элементами. Каждый элемент получает свой контроллер и свой слой (в режиме `svg-layer`).

### 4. Как обновить форму при изменении размера шрифта или внутреннего контента?

Библиотека автоматически отслеживает изменения размера элемента через `ResizeObserver`. Дополнительных действий не требуется.

### 5. Почему hover не срабатывает на мобильном устройстве?

Для сенсорных экранов библиотека эмулирует hover через события `touchstart` / `touchend`. Убедитесь, что:
- на триггерном элементе нет других обработчиков, блокирующих всплытие;
- в CSS нет правил, которые могут отменять эффект.

Если проблема сохраняется, включите `debug: true` и проверьте консоль.

### 6. Можно ли использовать библиотеку с фреймворками (React, Vue, Angular)?

Да. Просто вызовите инициализацию после монтирования компонента. Для динамически добавляемых элементов вызывайте `superellipseInit` после добавления в DOM.

**Пример для React:**

```jsx
useEffect(() => {
  const controller = elementRef.current.superellipseInit({ curveFactor: 1.2 });
  return () => controller.destroy();
}, []);
```

### 7. Как изменить форму суперэллипса анимированно?

Используйте `requestAnimationFrame` или библиотеку анимации (GSAP) с вызовом `setCurveFactor` на каждом кадре. Пример см. в разделе [examples.md](./examples.md).

### 8. Что означает константа G ≈ 0.5523?

Это значение `(4/3)*(√2-1)`, при котором кубические кривые Безье аппроксимируют четверть окружности. При `curveFactor = G` форма углов становится идеально круглой.

### 9. Можно ли использовать только генератор пути без подключения всей библиотеки?

Да, экспортируйте функцию `jsse_generateSuperellipsePath` и используйте её отдельно:

```js
import { jsse_generateSuperellipsePath } from 'superellipse.js';
const path = jsse_generateSuperellipsePath(200, 150, 30, 1.2);
```

### 10. Почему в консоли появляется предупреждение о CORS при чтении стилей?

Библиотека пытается проанализировать все таблицы стилей на странице для поддержки hover-триггеров. Если таблица стилей загружена с другого домена без заголовков CORS, браузер блокирует доступ. Это не критично – hover-триггеры будут работать только для стилей, которые можно прочитать.

Чтобы избежать предупреждения, разместите все стили на том же домене или добавьте атрибут `crossorigin` для внешних CSS.

### 11. Как отключить автоматическое обновление при изменении размеров?

Автоматическое обновление через `ResizeObserver` нельзя отключить (оно является частью функциональности). Если вам нужно временно приостановить обновления, можно вызвать `controller.disable()`, а затем `controller.enable()`.

### 12. Есть ли поддержка IE11?

Нет. Библиотека использует современные API (`WeakMap`, `ResizeObserver`, `MutationObserver`, `CSS.supports`), которые отсутствуют в IE11.

### 13. Почему при `curveFactor = 0` углы не становятся прямыми?

При `curveFactor = 0` форма представляет собой **скос** (прямые линии под углом 45°), а не прямые углы. Прямые прямоугольные углы получаются при `curveFactor = 2` (выгнутые) или `curveFactor = -2` (вогнутые). Если нужны обычные прямые углы без скругления, установите `border-radius: 0` и `curveFactor: 0`.

### 14. Как получить текущий путь для использования в других SVG?

Используйте метод `getPath()` контроллера:

```js
const pathData = controller.getPath();
document.querySelector('#other-svg path').setAttribute('d', pathData);
```

### 15. Влияет ли библиотека на производительность при скролле?

Библиотека обновляет форму только при изменении размеров, стилей или `curveFactor`. Во время скролла перерисовка не происходит, если только скролл не вызывает изменение размеров элемента (например, при скролле внутри элемента с `overflow: auto`). В режиме `svg-layer` может быть небольшая нагрузка на GPU из-за рендеринга SVG, но в целом она незначительна.

## Известные проблемы и пути решения

| Проблема | Решение |
|----------|---------|
| В Safari тени в режиме `svg-layer` могут отображаться с задержкой | Обновите Safari до последней версии. Используйте `clip-path` если тени не критичны. |
| После изменения класса элемента форма не обновляется | Библиотека отслеживает изменения атрибута `class` через `MutationObserver`. Убедитесь, что элемент не удаляется и не создаётся заново. |
| В Firefox наблюдается мерцание при анимации `curveFactor` | Используйте `precision: 2` (по умолчанию) и избегайте слишком частых обновлений (например, 60 FPS). |
| Hover-триггеры не работают для селекторов с `:has()` в старых браузерах | Библиотека автоматически переключается на fallback-режим (обход DOM). Производительность может быть ниже, но функциональность сохраняется. |
