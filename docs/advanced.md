# Продвинутые возможности

## Поддержка `:hover` и CSS-триггеров

Библиотека автоматически анализирует CSS-правила страницы и реагирует на изменение стилей при наведении (и других псевдоклассах), чтобы пересчитывать форму суперэллипса.

### Как это работает

1. При активации режима библиотека парсит все таблицы стилей (`document.styleSheets`).
2. Извлекает селекторы, которые могут влиять на целевой элемент (содержат `:hover`, соседние комбинаторы `+` или `~`).
3. Находит элементы-триггеры, при наведении на которые изменяются стили целевого элемента.
4. Регистрирует обработчики `mouseenter`/`mouseleave` (и `touchstart`/`touchend` для сенсорных экранов).
5. При срабатывании триггера запускается перерасчёт пути и обновление стилей.

### Поддерживаемые сценарии

#### 1. Простой hover на самом элементе

```css
.card:hover {
  border-radius: 40px;
}
```

Библиотека пересчитает суперэллипс при наведении на `.card`.

#### 2. Hover на родителе

```css
.container:hover .card {
  border-radius: 20px;
}
```

При наведении на `.container` обновится форма `.card`.

#### 3. Соседние селекторы (`+` и `~`)

```css
.button:hover + .card {
  border-radius: 50px;
}

.item:hover ~ .card {
  border-radius: 30px;
}
```

Библиотека корректно обрабатывает комбинаторы `+` (соседний элемент) и `~` (все последующие соседние элементы).

#### 4. Сложные селекторы с `:has()` (при поддержке браузера)

```css
.parent:has(.trigger:hover) .card {
  border-radius: 60px;
}
```

Если браузер поддерживает `:has()`, библиотека использует его для эффективного поиска триггеров. В противном случае применяется fallback-механизм (ручной обход DOM).

### Требования для корректной работы hover

- CSS-правила должны быть доступны для чтения (без CORS-ограничений). При использовании `file://` протокола могут возникнуть проблемы – используйте локальный сервер.
- Псевдокласс `:hover` должен присутствовать в селекторе, который влияет на `border-radius` (или другие управляемые свойства) целевого элемента.
- Изменение `border-radius` должно быть достаточно значительным, чтобы повлиять на форму суперэллипса.

### Отладка hover-триггеров

Включите режим отладки:

```js
const controller = element.superellipseInit({ debug: true });
```

В консоли будут отображаться сообщения о регистрации триггеров, входах и выходах из hover:

```
[JSSE] [STYLESHEET] [TARGET] [INIT]
[JSSE] [TRIGGER] [EVENT] true .container:hover .card
[JSSE] [HOVER] [IN] .container:hover .card MouseEvent
[JSSE] [MUTATION] [START]
[JSSE] [MUTATION] [UPDATE]
```

## Событийная модель

Контроллер генерирует события, на которые можно подписываться.

### Событие `update`

Возникает при любом обновлении (изменение размера, стилей, `curveFactor` и т.д.).

```js
controller.on('update', (e) => {
  console.log('Тип обновления:', e.data.type);
  // type может быть: 'full', 'styles', 'size', 'curveFactor'
});
```

### Событие `activate`

Возникает при активации режима.

```js
controller.on('activate', (e) => {
  console.log('Активирован режим:', e.data.mode);
});
```

### Событие `deactivate`

Возникает при деактивации режима.

```js
controller.on('deactivate', (e) => {
  console.log('Деактивирован режим:', e.data.mode);
});
```

### Событие `error`

Возникает при ошибках в обработчиках событий (не критические).

```js
controller.on('error', (e) => {
  console.error('Ошибка:', e.data);
});
```

## Отладка

### Включение отладки

При инициализации передайте опцию `debug: true`:

```js
const controller = superellipseInit('.element', { debug: true });
```

### Что выводится в консоль

- `[JSSE] [DEBUG]` – подтверждение включения отладки для элемента.
- `[JSSE] [CONTROLLER]` – настройки, статус, мутации.
- `[JSSE] [MODE]` – активация/деактивация режима, обновления.
- `[JSSE] [STYLESHEET]` – загрузка CSS-правил, поиск триггеров.
- `[JSSE] [HOVER]` – входы и выходы из hover-триггеров.
- `[JSSE] [MUTATION]` – обнаружение изменений, старт/конец пересчёта.

### Пример отладочного вывода

```
[JSSE] [DEBUG] true 
  {element: div.card}
[JSSE] [CONTROLLER] [SETTINGS] {mode: 'svg-layer', debug: true, curveFactor: 0.5523, precision: 2}
[JSSE] [MODE] activate(svg-layer)
[JSSE] [STYLESHEET] [LOADED]
[JSSE] [STYLESHEET] [TARGET] [INIT]
[JSSE] [TRIGGER] [EVENT] true .card:hover
[JSSE] [MUTATION] [DETECT] flow
[JSSE] [MUTATION] [START]
[JSSE] [MUTATION] [UPDATE]
[JSSE] [MUTATION] [END]
```

## Кэширование стилей

Библиотека использует глобальные хранилища для оптимизации:

### `jsse_styles` (WeakMap)

Кэширует вычисленные стили для каждого элемента. Позволяет избежать повторного чтения `getComputedStyle` при отсутствии изменений.

### `jsse_reset_css`

Хранит глобальные CSS-правила для сброса стилей режимов. Элементы `<style>` добавляются в `<head>` один раз для каждого режима и удаляются, когда больше нет активных контроллеров, использующих этот режим.

### `jsse_controllers` (WeakMap)

Связывает DOM-элементы с их контроллерами. Позволяет получать контроллер через `element.superellipse`.

## Интеграция с анимациями

### Анимация `curveFactor`

```js
const controller = element.superellipseInit({ curveFactor: -2 });

let factor = -2;
function animate() {
  factor += 0.02;
  if (factor > 2) factor = -2;
  controller.setCurveFactor(factor);
  requestAnimationFrame(animate);
}
animate();
```

### Анимация размера элемента

Библиотека автоматически отслеживает изменения размера через `ResizeObserver` и обновляет форму. При анимации `width`/`height` на CSS-переходах форма будет плавно меняться (если браузер поддерживает анимацию `clip-path`).

### Анимация `border-radius`

```css
.element {
  transition: border-radius 0.3s ease;
}
.element:hover {
  border-radius: 50px;
}
```

При наведении суперэллипс плавно изменит форму (если анимируется свойство `clip-path` или перерисовывается SVG).

## Работа с сенсорными экранами

Для корректной обработки hover на сенсорных экранах библиотека регистрирует глобальный обработчик `touchend`. При касании триггера срабатывает `touchstart` → `mouseenter` (эмуляция). Отпускание касания вызывает `touchend` → `mouseleave`.

Это позволяет использовать hover-эффекты на мобильных устройствах (обычно они работают по первому касанию).

## Производительность

### Советы по оптимизации

1. **Используйте `clip-path`** для большого количества элементов (например, списки, галереи).
2. **Уменьшите `precision`** до 1 или 0 для очень динамичных анимаций.
3. **Избегайте частых изменений `border-radius`** через CSS-анимации – лучше анимировать `curveFactor`.
4. **Не включайте `debug`** в production.
5. Для `svg-layer` ограничьте количество одновременно анимируемых элементов.

### Влияние `precision` на производительность

| precision | Размер пути | Качество | Производительность |
|-----------|-------------|----------|---------------------|
| 0 | ~200 символов | Грубое | Отличная |
| 2 | ~600 символов | Хорошее | Хорошая |
| 4 | ~1000 символов | Отличное | Средняя |

## Доступ к глобальным хранилищам (для расширенного использования)

Хотя эти объекты не являются частью публичного API, они доступны в сборке UMD как свойства глобального объекта `Superellipse`:

```js
// Только в UMD-версии
const { jsse_controllers, jsse_styles, jsse_counter } = window.Superellipse;
```

**Не рекомендуется** модифицировать их напрямую – это может нарушить работу библиотеки.
